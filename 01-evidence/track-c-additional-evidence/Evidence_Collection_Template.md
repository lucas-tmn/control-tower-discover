# Track C — Additional Evidence Collection Template

## Purpose
Collect the evidence that lives outside Power BI entirely — SOPs, Excel
files, email processes, shadow processes, manual workflows — named by
interviewees in Track B (field B8, Shadow tools) or discovered independently.

## Why this track exists
Track A shows what reports display. Track B shows what people say they do.
Track C is the check against both: does a written SOP match what the report
implies and what the interview claimed? Discrepancies here are some of the
highest-value discoveries — they usually mean the "official" process and the
"real" process have quietly diverged.

## Intake log (one row per artifact collected)

| Field | What to capture |
|---|---|
| Artifact name | e.g. "Safety Stock Override Process.xlsx" |
| Source / owner | Who maintains it, which team |
| Format | SOP doc · Excel · email thread · Slack channel · meeting cadence |
| Related decision(s) | Link to Decision Registry ID(s) if known |
| Named by | Which interview or report surfaced this |
| Governed? | Yes (owned, versioned, backed up) / No (personal file, undocumented) |
| Last updated | Date, if known |
| Corroborates or conflicts | Does it match Track A/B findings, or contradict them? |
| Notes | Anything surprising |

## Collection process
1. **After each Track B interview**, request any artifact named under
   Shadow tools (B8) directly — don't wait for a separate outreach round.
2. **Store artifacts in a governed location** (not scattered downloads) —
   per Repository Principle "No empty scaffolding" / "Repository follows
   Knowledge," only artifacts that are actually reviewed get logged; a pile
   of unread files is not evidence yet.
3. **Tag governed vs ungoverned** immediately — this feeds Data Readiness
   (D1) in the registry directly. An SOP that only exists as an email
   attachment from 2023 is itself a finding, not just a source.
4. **Flag conflicts loudly.** If an SOP says one threshold and the report
   logic uses another, log it as an Open Question (F4) against the
   relevant decision — do not silently pick one.

## What "done" looks like for Track C
Not: every possible document collected. Rather: for each decision under
active discovery in Sprint 2, at least one attempt has been made to find
its non-Power-BI evidence, and the result (found + reviewed / found but not
reviewed / confirmed not to exist) is logged — so absence of SOP evidence is
a documented finding, not an oversight.
